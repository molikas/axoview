## Goal

End state: a codebase that's **sparkling clean** post-v1.0 — confirmed dead code deleted, audit claims made true (or honestly corrected), productization decisions resolved, external GitHub Settings actions completed. The artifact at [`docs/technical-review-2026-05.md`](../technical-review-2026-05.md) names the gaps; this tactical closes them.

**Why now:** the productization audit shipped v1.0.0 with a substantial deferred-debt tail. The "deferred-by-design" framing was honest but easy to lose the cumulative weight of. The technical review artifact made the weight visible; this tactical executes the cleanup before the debt metastasizes into v1.1+ work.

**Explicit non-goals:**
- New features. v1.1 may bring features; THIS tactical does not.
- Architectural refactors of large surfaces (Track 6 catalogues them; defers them).
- Closing the server-runtime test gap (Track 5 catalogues it; spawns its own tactical).
- Closing operational/security gaps that need real-user signal first (Express structured logging, etc.).
