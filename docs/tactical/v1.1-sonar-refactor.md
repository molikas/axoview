## Per-batch sub-task template (applied per PR)

For each PR (M2–M7), the executing agent follows this loop:

1. **Pre-edit investigation.** Read the target function in full. Identify the cognitive-complexity hot spots (nested branches, repeated conditional blocks, long lambda chains). Sketch the decomposition (extract-function / early-return / strategy-table / state-machine collapse) before editing.
2. **Locate the test coverage.** Identify which unit / contract / e2e tests exercise this function. If coverage gap exists, FILE AS FINDING and proceed — but flag in PR description as a future test row.
3. **Refactor in N micro-commits.** Each micro-commit: one extraction or one early-return restructure. Run `yarn build` + the relevant test suite after each micro-commit. Conventional-commits subject: `refactor(<scope>): <what>` (NOT `fix:` — behavior is preserved).
4. **Verify Sonar score.** After the final micro-commit, re-run `npx sonar-scanner` (or document that the local-Sonar re-run is deferred to next user inventory pass). The refactored function MUST drop below 15 cognitive complexity.
5. **Manual smoke (per file's interaction surface).** Use the per-file smoke checklist below.
6. **Open PR.** PR description: links Tier 1 row number(s); cites the cognitive-complexity before/after; lists smoke steps performed; flags any findings.
7. **Pause for user review.** User merges; orchestrator re-consults for next batch.
